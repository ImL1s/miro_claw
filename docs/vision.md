# MiroClaw — 去中心化群體智能預測協議

> MiroFish × OpenClaw = 去中心化的群體智能預測協議

---

## Vision

**每個 OpenClaw 節點既是 AI Agent，又是區塊鏈驗證者。多個節點協力跑同一場多智能體模擬，用鏈上存證讓預測不可篡改，用信譽系統讓好的預測者被看見。**

---

## 架構決策

| 決策項 | 結論 |
|:---|:---|
| 推演引擎 | MiroFish（OASIS 模擬 + GraphRAG + Report Agent） |
| Agent 網路 | OpenClaw Gateway Protocol v3 |
| 共識層框架 | **Cosmos SDK**（Go 語言、完全主權、IBC 跨鏈、零 gas fee） |
| 區塊鏈定位 | **獨立可選模組**（與群體智能推演分開，但可方便互接） |
| 經濟模型 | **預測存證 + 信譽系統**（非賭博，無 Polymarket 模式） |
| LLM 成本 | **Gateway 共用**（MiroFish 直接指向 Gateway 的 LLM endpoint） |
| P2P 模式 | **分散式模擬**（多節點協力跑同一場模擬，各出一份 Agent） |

## 用戶體驗流程

```
1️⃣  對 OpenClaw 說「推演聯準會降息影響」
2️⃣  Gateway 提取種子 → 呼叫 MiroFish API
3️⃣  背景模擬（55 Agent × 40 輪）
     └─ 多人模式：Agent 分配到各節點並行跑
4️⃣  推播通知 → Canvas 載入視覺化結果
     └─ AI 用 A2UI 高亮關鍵事件
5️⃣  Chat 追問 → 採訪模擬中的 Agent
6️⃣  結果 hash 上鏈存證（可選）
7️⃣  事後驗證 → 信譽分數更新
```

## 分散式模擬（核心創新）

### 單機 vs 分散式

```
單機：55 Agent 全在 Node A → 成本 100%

分散式（5 節點）：
  Node A → Agent  1-11  → 成本 20%
  Node B → Agent 12-22  → 成本 20%
  Node C → Agent 23-33  → 成本 20%
  Node D → Agent 34-44  → 成本 20%
  Node E → Agent 45-55  → 成本 20%
  
  每輪結束 → 廣播行為 → 同步後開始下一輪
```

### OASIS 改造項

| 模組 | 改動 |
|:---|:---|
| 模擬循環 | 拆為「Agent 執行」+「跨節點同步」兩階段 |
| 行為廣播 | libp2p gossip |
| 容錯 | 節點掉線 → 其他節點接手 Agent |
| 驗證 | 鏈上存每輪 Agent 行為 hash |

## 經濟模型（非賭博）

```
貢獻算力 ─→ 信譽分
預測準確 ─→ 信譽加倍
信譽高 ─→ 訂閱者多 ─→ 收入
```

所有鏈上操作零手續費。

## 設計原則

> **群體智能推演** 和 **區塊鏈共識** 是兩個獨立模組。
> 各自可以單獨運作，但設計上方便互相接入。

- 只跑推演不上鏈？✅ 完全可以
- 只跑鏈不跑推演？✅ 也可以（但沒什麼意義）
- 兩者結合？✅ 推演結果自動存證上鏈

## 冷啟動策略

1. **自己先跑一個節點**（Gateway + MiroFish，單機驗證全流程）
2. 跑通後拉 2-3 個人測 P2P 推演分享
3. 分散式模擬需要 3+ 節點才有意義
4. Cosmos 驗證者最少 4 個節點即可啟動
